# 记忆层设计

## 1. 记忆层定位

记忆层保存用户长期投资上下文，不保存或替代事实行情数据。

它回答的问题是：

- 用户为什么买这只股票？
- 上次复盘时担心什么？
- 用户偏好高股息还是成长？
- 某个持仓的撤出条件是什么？
- 上次 Agent 给出的观察点是什么？

## 2. 记忆类型

### 2.1 UserPreference

用户偏好。

字段：

- 风险承受能力；
- 偏好市场：A股/港股；
- 偏好风格：高股息/成长/价值/周期；
- 最大单股仓位；
- 禁止行业；
- 关注行业。

### 2.2 PortfolioMemory

持仓记忆。

字段：

- 股票代码；
- 买入理由；
- 预期持有周期；
- 核心投资假设；
- 撤出条件；
- 最近一次复盘结论。

### 2.3 ThesisMemory

投资假设记忆。

字段：

- 假设内容；
- 支持证据；
- 证伪条件；
- 状态：active / weakened / invalidated；
- 更新时间。

### 2.4 ReviewMemory

复盘记忆。

字段：

- 月度/季度/年度；
- 当期结论；
- 风险变化；
- 下次观察点；
- 当时数据快照 ID。

### 2.5 DecisionLog

决策日志。

字段：

- 决策日期；
- 动作：持有/观察/谨慎持有/减仓撤出/剔除；
- 决策原因；
- 证据链；
- 后续复盘结果。

## 3. 记忆写入原则

允许写入：

- 用户明确输入的偏好；
- 用户持仓理由；
- 用户确认的投资假设；
- 每次报告的摘要；
- 已生成的决策日志。

不允许写入：

- 未经确认的模型猜测；
- 编造的财务数据；
- 临时行情波动；
- 未验证新闻；
- API key 或隐私敏感信息。

## 4. 记忆使用原则

Agent 使用记忆时必须区分：

- 事实数据：来自 Data/Tools；
- 用户上下文：来自 Memory；
- 模型推理：来自 Agent。

报告里必须避免把用户偏好写成市场事实。
