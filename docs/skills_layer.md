# Skills 层设计

## 1. Skills 是什么

Skills 是一组可复用的研究流程。它不同于 Tools：

- Tool 是一个动作；
- Skill 是一套流程；
- Agent 可以调用 Skill；
- UI 也可以直接触发 Skill。

## 2. 核心 Skills

### 2.1 monthly-holding-review

月度持仓体检。

输入：

- 用户持仓列表；
- 分析月份；
- 用户风险偏好；
- 是否调用 LLM。

流程：

1. 查询持仓；
2. 查询 250 日行情表现；
3. 查询风险事件；
4. 查询上次投资假设；
5. 计算长线评分变化；
6. 输出持有/谨慎持有/减仓撤出建议；
7. 生成月报。

### 2.2 quarterly-earnings-review

季度财报复盘。

重点判断：

- 营收与利润是否符合预期；
- 毛利率/净利率是否恶化；
- 经营现金流是否支持利润；
- 资产负债率是否异常；
- 投资假设是否被证伪。

### 2.3 annual-universe-rebuild

年度股票池重建。

流程：

1. 全市场股票扫描；
2. 风险剔除；
3. 分类；
4. 因子评分；
5. 行业排名；
6. 生成重点股票池；
7. 复盘上一年度规则有效性。

### 2.4 single-stock-longhold-analysis

单股长线分析。

输出：

- 公司概览；
- 财务质量；
- 成长性；
- 估值；
- 市场趋势；
- 风险；
- 多头观点；
- 空头观点；
- 风控裁决。

### 2.5 risk-first-screening

风险优先筛选。

先排雷，再分析。

剔除：

- ST / 退市风险；
- 长期亏损；
- 非标审计；
- 现金流严重背离；
- 流动性不足；
- 财务异常；
- 治理风险。

## 3. Skill 文件规范

每个 Skill 都应有：

- `*.skill.md`：面向 Agent 的流程说明；
- `*.ts`：可执行编排函数；
- `*.test.ts`：测试样例；
- 输入输出 schema。
